# Telegram Bot — Deploy & Operate

The bot (`bot/worker.js`, Cloudflare Worker) ingests playbills sent via Telegram — photo/scan,
PDF, link, free text, or live location ("what am I watching right now") — parses them with the
Claude API into the performance schema, shows a preview with «вы уже встречали …» cross-links,
and on operator confirmation commits `items/` + `inventory/` (+ `playbills/` scan) straight to
`main`. The push triggers the normal Pages rebuild. The bot reads its people-memory from the
published site's `/data/index.json` (generated by `helpers/build_site.py`) — no database.

**Rules of the road:**
- The bot writes **project data only** (items, inventory rows, playbill scans) and **only after
  an explicit operator confirmation tap** — never project mind, never sessions.
- Bot commits are prefixed `bot:`; the commit itself is the record of the ingestion. The next
  human session's retrospective covers accumulated bot ingestions.
- All credentials live in Cloudflare Worker secrets — credential, do not expose: bot token
  (from @BotFather), webhook secret (self-chosen random string), Anthropic API key
  (platform.claude.com), GitHub fine-grained PAT (contents read/write on this repo only).
  Nothing secret is committed here.
- Venue coordinates: canonical in `inventory/venues.md`; `bot/worker.js` `VENUES` is a copy —
  update both.

## One-time setup (operator)

1. **Bot:** message @BotFather → `/newbot` → save the token.
2. **Your Telegram ID:** message @userinfobot (or similar) → put the numeric ID into
   `bot/wrangler.toml` → `OPERATOR_CHAT_ID`.
3. **Cloudflare:** free account at dash.cloudflare.com, then locally:
   ```
   cd bot
   npx wrangler login
   npx wrangler kv namespace create PENDING     # → paste the id into wrangler.toml
   npx wrangler secret put TELEGRAM_BOT_TOKEN
   npx wrangler secret put TELEGRAM_WEBHOOK_SECRET   # invent a long random string
   npx wrangler secret put ANTHROPIC_API_KEY
   npx wrangler secret put GITHUB_TOKEN              # fine-grained PAT: this repo, Contents RW
   npx wrangler deploy                               # → note the workers.dev URL
   ```
4. **Webhook:** point Telegram at the worker (same secret as above):
   ```
   curl "https://api.telegram.org/bot<TOKEN>/setWebhook" \
     -d "url=https://<worker>.workers.dev/webhook" \
     -d "secret_token=<TELEGRAM_WEBHOOK_SECRET>"
   ```
5. **Smoke test:** send the bot `/start`, then a mariinsky.ru event link — expect a preview
   with confirm buttons; tap ✅ and check the repo for a `bot:` commit and the site for the
   new performance ~1 min later.

## Operating notes

- Model: `MODEL` var in `wrangler.toml` (`claude-opus-4-8` default; `claude-haiku-4-5` is ~5×
  cheaper and usually fine for clean scans). Redeploy after changing vars.
- Location flow currently knows only the Mariinsky's three buildings; extend
  `inventory/venues.md` + `VENUES` in the worker when a new theatre appears.
- Bot code changes are **project mind** — session + operator approval, then `npx wrangler deploy`.
- If parsing goes wrong after a confirm: items are editable (only sessions are immutable) —
  fix `items/<file>.md` in the next session and note it in that session's log.
